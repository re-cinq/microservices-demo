# Implementation Plan: Browser-Side Product Search

**Branch**: `attendee/joel-sanmoogan` | **Date**: 2026-05-11 | **Spec**: [spec.md](./spec.md)

## Summary

Shoppers on the Online Boutique home page have no way to narrow the product list without scrolling. This plan delivers a client-side search box above the product grid in `home.html` that filters visible product cards by name on every keystroke. All filtering is done in the browser against the server-rendered HTML; no backend, no new service, no new dependency, no build-pipeline change.

## Design

### 1. `home.html` — Search input markup

Insert immediately above the existing `<div class="col-12"><h3>Hot Products</h3></div>` row:

```html
<div class="col-12 product-search-box">
  <input
    type="search"
    id="product-search-input"
    class="form-control"
    placeholder="Search products…"
    aria-label="Search products"
    oninput="filterProducts(this.value)"
  />
</div>
```

Add an empty-state message inside the grid container, after the `{{ range $.products }}` block:

```html
<p id="no-results-msg" style="display:none;">No products match your search.</p>
```

### 2. `home.html` — Inline filter script

Insert before `{{ end }}` (end of the `home` template define block):

```html
<script>
function filterProducts(query) {
  var term = query.trim().toLowerCase();
  var cards = document.querySelectorAll('.hot-product-card');
  var visible = 0;
  cards.forEach(function(card) {
    var name = card.querySelector('.hot-product-card-name');
    var match = !term || (name && name.textContent.trim().toLowerCase().indexOf(term) !== -1);
    card.style.display = match ? '' : 'none';
    if (match) visible++;
  });
  document.getElementById('no-results-msg').style.display = visible === 0 && term ? '' : 'none';
}
</script>
```

### 3. `styles.css` — Search box styles

Append to `src/frontend/static/styles/styles.css`:

```css
/* Product search box (browser-side filter) */

}

.product-search-box input[type="search"] {

  width: 100%;

  font-size: 1rem;
}
```

## Acceptance Criteria Traceability

| Spec Requirement | Design Element |
|---|---|
| FR-001: search input visible without scrolling | `.product-search-box` div above the product grid heading |
| FR-002: list updates on every input change | `oninput="filterProducts(this.value)"` fires on each keystroke |
| FR-003: case-insensitive matching | `.toLowerCase()` applied to both query and card name |
| FR-004: empty/whitespace input shows all products | `var term = query.trim().toLowerCase()` — empty string is falsy → all shown |
| FR-005: zero-match empty-state message | `#no-results-msg` toggled when `visible === 0 && term` |
| FR-006: no network requests when typing | Filtering operates on existing DOM; no fetch/XHR |
| FR-007: keyboard-accessible | Standard `<input>` in document flow; reachable via Tab |
